Destructuring (arrays, objects) 

Destructuring allows you to unpack values from arrays or objects into individual variables â€” instead of accessing them one by one. 

Why Destructuring is Useful:

Traditional way:

With destructuring:

âœ… write less code and itâ€™s easier to read. 

 

2ï¸âƒ£ Easier access to nested data

  info: { 
    city: "Chennai", 
    zip: 600001 
  }

// Traditional

âœ… Without destructuring, have to write extra if statements to check for missing data.

2) Spread and Rest Operator (...) 

Both use the same syntax (...), but meaning changes by where you use it. 

Spread (expands values) 

It spreads array or object elements into individual items. 

Example (Array): 

const arr1 = [1, 2]; 
const arr2 = [3, 4]; 
const result = [...arr1, ...arr2]; 
console.log(result); // [1, 2, 3, 4] 
 

Example (Object): 

const user = { name: "Ram" }; 
const info = { age: 25 }; 
const merged = { ...user, ...info }; 
console.log(merged); // { name: 'Ram', age: 25 } 
 

Copy arrays (without affecting original): 

const copy = [...arr1]; 

 

Rest (collects values) 

It collects remaining values into an array or object. 

Example (Array): 

const [a, ...rest] = [1, 2, 3, 4]; 
console.log(a);    // 1 
console.log(rest); // [2, 3, 4] 

 

Output: 

1 
[ 2, 3, 4 ] 

 

Used in function parameters or destructuring to gather remaining values. 

function sum(...numbers) { 
  console.log(numbers); // [1, 2, 3, 4] 
  return numbers.reduce((a, b) => a + b); 
} 
 
console.log(sum(1, 2, 3, 4)); // 10 

 

Output: 

[ 1, 2, 3, 4 ] 
10 

 
 

ðŸ”¹ Rest in destructuring: 

const [first, ...rest] = [10, 20, 30, 40]; 
console.log(first); // 10 
console.log(rest);  // [20, 30, 40] 

 

 

 

Template Literals (Backticks ``) 

 

Used for string interpolation (inserting variables easily) and multi-line strings. 

 

 

Before (old way): 

const name = "Ram"; 
const msg = "Hello " + name + ", welcome!"; 
console.log(msg); 

 

Output:

using template literals: 

const name = "Ram"; 
const msg = `Hello ${name}, welcome!`; 
console.log(msg); 

Output: 

Hello Ram, Welcome 

 
 

Supports multiple lines: 

Multiline strings 

Before ES6: 

const text = "Line 1\nLine 2\nLine 3"; 
 

With template literals: 

const text = ` 

This is line 1 

This is line 2 

`; 

console.log(text); 

 

Embed expressions 

const a = 5; 
const b = 10; 
console.log(`Sum is ${a + b}`); // Sum is 15 
 

âœ… You can compute values directly inside the string. 

 

4) Modules (import / export) 

Used to split code into separate files and reuse them. 

1ï¸âƒ£ Named Exports (Recommended for multiple functions) 

// math.js 
export const add = (a, b) => a + b; 
export const sub = (a, b) => a - b; 
 

Each function is exported individually. 

Then in another file, you can import like:

console.log(add(2, 3)); // 5 
console.log(sub(5, 2)); // 3 
 

 

2ï¸âƒ£ Export all at once (Named Export)

const sub = (a, b) => a - b; 
 
export { add, sub }; 
 

âœ… Works the same as above. 

 

3ï¸âƒ£ Default Export (Optional, usually for one main thing)

const math = { 
  add: (a, b) => a + b, 
  sub: (a, b) => a - b, 
}; 
 
export default math; 
 

Then import like: 

import math from './math.js'; 
 
console.log(math.add(2, 3)); // 5 
console.log(math.sub(5, 2)); // 3
